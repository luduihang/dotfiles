# 跨场景语音转写文本规整 Skill

> 适配海外ASR接口繁简混出 + 全品类口述文稿，全文体通用（科研复盘、日常记事、情绪随笔、抒情散文、琐事记录、工作沟通手记）

## 一、核心顶层铁律（优先级最高，不可突破）

1. **完全尊重原文行文结构**，禁止自主拆分、新增、合并段落与文章框架：
   - 原文自带01/02/03层级、小标题，原样保留不动
   - 原文无任何分级、无分段、散文随性连贯行文，绝不主动帮用户划分条目、拆分板块、归纳大标题、梳理逻辑框架
   - 原有段落排布、语句先后顺序 100% 沿用作者原生排版
   - 无论内容是工作干货还是感性散文，不做内容结构化梳理、不额外搭建目录体系

2. **只做文字表层规范化**，绝不主观改写语义、调整作者表达逻辑、补充内容、删减观点与心理描写，抒情文案的语感、随笔随性文风完整保留

3. **适配四大痛点**：境外语音API随机穿插繁体、语音口述同音错别字、口语助词堆砌、全篇缺少标点/语句粘连

## 二、分步处理流水线（固定执行顺序）

### Step1：繁简统一处理（海外API专属规则）

- 全文繁体汉字统一转为简体中文，解决简繁混杂问题
- 自建专有名词白名单：人名、外文术语、品牌、专业名词（如DeepSeek、Claude、莫里斯、各类人名）不参与繁简转换、不随意修改用字
- 人名、小众专有名词不在白名单内时，按通用简体转换；出现歧义则正文保留原字，在文末备注

### Step2：语音同音错字/口误修正

- 依托上下文修正语音识别产生的同音错别字（如：灵道→灵敏、迷茅→迷茫等仅作为案例参考）
- 无法通过上下文确认的疑似错字、生造口语用字，正文保留原文不擅自修改，统一录入文末修正备注清单
- 仅修正识别失误用字，不替换作者刻意使用的口语化用词、个性化表达、散文文艺化措辞

### Step3：标点规范化补正

- 按照现代汉语通用标点规范，为无标点长句补齐逗号、句号、顿号；修正错用、多余标点
- 不依靠标点自主拆分段落，原文哪里换行就保持原有换行，散文大段连写仍保持整段格式，仅优化标点

### Step4：冗余口语词轻量化精简（克制优化）

- 剔除无实际语义、重复堆砌的口头衔接词：重复冗余的「然后、那么、啊、吧」，同位置连续堆叠的连接词按需精简
- 只删无效赘词，不改动原句语序、不重组句式
- 作者刻意保留的抒情语气、随笔口语化表达（散文、情绪文案里的生活化语气词）予以保留，不强行书面化
- 严禁大面积改写句子、压缩文意、润色升华文案

### Step5：排版最终固化

- 原文分段、换行、序号、空行全部原样继承
- 散文无框架、无分级，输出后依旧保持原生散漫行文结构，不新增任何层级标题与编号

## 三、绝对禁止行为清单

- ❌ 禁止AI自动生成01、02、03、一级标题、分类小标题等任何框架内容
- ❌ 禁止根据内容逻辑拆分大段文字、重新排版分段、归纳内容模块
- ❌ 禁止区分文体差异化重构行文，工作文/抒情散文使用同一套规整标准
- ❌ 禁止修改作者主观想法、心理感悟、情感表述、写作立意

## 四、输出格式规范

1. **主体正文**：经繁简、错字、标点、冗余助词优化后的文稿，排版结构和原文一一对应
2. **文末固定附加【修改备注】**：逐条列明：
   - ① 繁简转换位置
   - ② 已修正错别字
   - ③ 存疑未改动原文用字
   - 无修改则备注：全文无格式与文字修正

## 五、一键调用指令

启用【通用语音口述原稿标准化修正Skill】，严格遵循技能铁律，不自主划分文章框架、不拆分段落结构，按流水线规则处理下方文本，输出规整正文+文末修改备注：

```
【粘贴原文内容】
```

## 六、使用示例

### 示例1：散文无框架示范

**原始口述文本（生活随笔）：**
> 今天傍晚出门散步然后晚风很舒服吧走在街边的时候心里莫名放空啊很多积攒的烦心事一瞬间就散开了那么路边的树影摇摇晃晃灵道很治愈

**修正输出：**
> 今天傍晚出门散步，晚风很舒服，走在街边的时候心里莫名放空，很多积攒的烦心事一瞬间就散开了，路边的树影摇摇晃晃，很治愈。
>
> 【修改备注】原文"灵道"为语音识别错误，结合上下文修正为"很"；全文繁体无。

### 示例2：带原生编号原文

**原始口述文本：**
> 01、日常感悟 然后晨起喝咖啡吧阳光透过窗子落在桌面那么思绪慢慢沉静下来

**修正输出：**
> 01、日常感悟 晨起喝咖啡，阳光透过窗子落在桌面，思绪慢慢沉静下来。
>
> 【修改备注】删除冗余语气词，无错字、繁简改动。
